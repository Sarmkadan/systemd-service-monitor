# Contributing to systemd-service-monitor

Thank you for your interest in contributing!

## How to Contribute

1. **Fork the repository** on GitHub.
2. **Create a branch** for your feature or fix.
3. **Write tests** for any changes you make.
4. **Ensure the build passes** by running `dotnet build` and `dotnet test`.
5. **Submit a Pull Request** to the `main` branch.

## Standards

- Follow the existing coding style (see `.editorconfig`).
- Ensure all public APIs have XML documentation.
- Maintain consistency with the existing project structure.

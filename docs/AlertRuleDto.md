# AlertRuleDto

Represents a rule that defines conditions for triggering alerts in the system monitoring service. Used to configure alerting behavior for service health checks and other monitored conditions.

## API

### Properties
- **`Id`** (string, required)
  Unique identifier for the alert rule. Used to reference the rule in other operations.

- **`Name`** (string, required)
  Human-readable name for the alert rule. Should be descriptive of the condition being monitored.

- **`Description`** (string, optional)
  Additional context about the alert rule. May include details about monitoring scope or expected behavior.

- **`IsEnabled`** (bool, required)
  Indicates whether the alert rule is active and should generate alerts when conditions are met.

- **`Severity`** (string, required)
  Severity level of alerts generated by this rule. Valid values are defined by the system (e.g., "Warning", "Critical").

- **`Condition`** (string, required)
  JSON-formatted string defining the monitoring condition that triggers the alert. Structure is system-specific.

- **`CreatedAt`** (DateTime, required)
  Timestamp when the rule was created. Set by the system on creation.

- **`UpdatedAt`** (DateTime, optional)
  Timestamp when the rule was last modified. Null if never updated.
